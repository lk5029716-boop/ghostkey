What changed (and why)

Fix #1 — splash
  • Before: home was OnboardingScreen / PinScreen directly. On cold launch the user saw a brief blank; on warm launch the brand never appeared.
  • After: home: SplashScreen(...) always shows for 1.5s with the real brand logo (Canvas1.png) + scale-in + pulsing glow ring.

Fix #3 — second-time hang
  • Before: await BrandIconRegistry.init(); await CodeStore.init(); ran serially in main(). On warm start, SQLite reopen + 1500+ SVG icons froze the first frame.
  • After: await Future.wait([...]) — both run in parallel. Splash covers the wait so the user never sees a blank.
  • Settings biometric toggle now actually persists via BiometricService.setEnabled, with a snackbar if the device has no fingerprint enrolled.

Fix #2 — real biometric
  • Before: the "Biometric Lock" toggle in Settings was a bool field that did nothing. The fingerprint icon in the unlock screen was a Material icon, not a real prompt.
  • After: toggle persists via BiometricService. Unlock screen calls LocalAuthentication.authenticate() for real. The OS dialog (BiometricPrompt on Android) appears automatically when the user reaches the unlock screen.
  • Hidden fix: MainActivity was extending FlutterActivity, which cannot host BiometricPrompt. Switched to FlutterFragmentActivity — without this, the real prompt would never have shown.
